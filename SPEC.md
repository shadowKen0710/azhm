# 阿兹海默症患者守护 App — 产品规格说明（SPEC）

> 版本：POC v0.1 · 日期：2026-06-30
> 本文档由访谈结论沉淀而成，是开发与验收的唯一事实来源。先实现，再依此逐项验收。

---

## 0. 决策摘要（访谈结论）

| 维度 | 结论 |
|---|---|
| 主要用户 | **照护者监控为主**（健康/安全），**患者**可听 AI 模拟家人声线并与之对话 |
| 核心能力 | ①用药/日程提醒 ②一键 SOS 呼救 ③记忆辅助/认人卡 ④AI 仿家人声线对话 |
| 端形态 | **两套 UI**：照护者控制台（**竖屏手机**）+ 患者「老人屏」（**横屏智能屏/电视大屏**），通过角色切换预览 |
| 患者交互模型 | **零操作被动接收**：患者大屏固定摆放，家人来电由照护者远程发起、AI 仿真来电按日程自动发起，患者侧一律**自动接通**；同时保留患者主动点家人头像拨出（并重） |
| 照护者 | **单一主照护者**（不做家庭群组/多级声线） |
| 数据来源 | **假数据 + 预留 API 层**（service 层抽象，可无痛替换真实后端） |
| 技术形态 | **Web 应用（Vite + React + TypeScript）** —— 由我推荐选定，最快出可点击原型，单/双端皆可演示 |
| 视觉基调 | **温暖疗愈、家的感觉**（两端统一暖调，患者屏更大字、更安心） |
| AI「持续学习」 | **仅做可视化占位**，不真实训练模型 |
| 明确不做 | 真实语音克隆/ASR/TTS、真实账号登录鉴权、推送/短信/电话集成（详见 §10） |

> 注：实时定位/电子围栏**未被选为核心能力**。SOS 场景需要展示「最后位置」时，仅用**静态 mock 地图/坐标占位**，不接真实 GPS 或地图 SDK。

## 1. 技术形态与架构

- **栈**：Vite + React 18 + TypeScript；路由用 React Router；状态用 React Context + hooks（POC 体量无需 Redux）。
- **样式**：CSS Modules 或 Tailwind（择一，全局暖色 design token，见 §3）。
- **音频**：患者对话播放**预置音频片段**或 Web Speech `SpeechSynthesis` 兜底；不接真实克隆/ASR。
- **目录约定**：
  - `src/services/` —— 数据访问层，所有页面只调 service，不直接碰 mock。每个 service 暴露 `getX() / updateX()` 返回 `Promise`，内部当前读 `src/mock/`，未来替换为 `fetch`/API client 即可。
  - `src/mock/` —— 假数据 JSON + 模拟延迟/失败开关（用于演示 loading/error）。
  - `src/surfaces/caregiver/` 与 `src/surfaces/patient/` —— 两端页面隔离。
  - `src/components/` —— 共享组件（状态占位、暖色卡片、大按钮等）。
- **角色切换**：顶部全局开关 `照护者 ⇄ 患者`，纯前端模拟，不做真实鉴权（见 §10）。
- **可注入的演示开关**（dev-only 面板）：强制 `loading / empty / error`、模拟患者「失联」、触发一条 SOS、快进到某提醒到点。用于验收时一键复现每种状态。

## 2. 信息架构与路由表

顶部持久化 **角色切换器**（照护者 / 患者）+ 演示控制台入口。

### 照护者控制台 `/caregiver/*`
| 路由 | 页面 | 核心任务 |
|---|---|---|
| `/caregiver` | 概览 Dashboard | 一眼看到患者当前状态、今日服药、最近情绪、待处理告警 |
| `/caregiver/reminders` | 提醒管理 | 增/删/改用药与日程提醒，看完成情况 |
| `/caregiver/alerts` | 告警 / SOS 中心 | 查看实时 SOS 全链路状态、未服药升级、失联告警、历史 |
| `/caregiver/memory-cards` | 认人卡管理 | 维护家人照片/姓名/关系/称呼 |
| `/caregiver/voices` | 声线管理 | 家人声线授权录制、列表、撤销；AI 学习进度占位 |
| `/caregiver/conversations` | 对话记录 | 回放患者×AI 对话摘要与情绪曲线 |
| `/caregiver/settings` | 设置 | 患者档案、主照护者信息、安全区占位 |

### 患者端「老人屏」 `/patient/*` —— 横屏大屏（智能屏/电视），固定摆放、零操作被动接收
| 路由 | 页面 | 核心任务 |
|---|---|---|
| `/patient` | 待机主页（横屏） | 大字问候+巨型时钟 · 今日提醒 · 巨大 SOS ；右侧家人头像可**主动拨出** ；被动**来电覆盖层**（家人/AI）倒计时**自动接通** |
| `/patient/talk/:voiceId` | 通话页（横屏） | 与家人通话；`?by=ai` 显示「AI 模拟声音」明示、`by=family` 为真人不显示；**环境聆听**指示（设备自动听，无按键）+ 大字幕 + 挂断 |
| `/patient/who` | 认人卡（横屏） | 大头像逐位浏览「这是你的女儿」+ 直达通话 |
| （覆盖层）来电 | 被动接听 | 「家人来电 / AI 模拟来电」+ 大头像 +「无需操作，N 秒后自动接通」，可选「以后再说」 |
| （覆盖层）SOS 确认 | SOS 倒计时 | 误触可撤销的倒计时确认 |

> 触发方式：家人来电由**照护者控制台「陪伴 → 远程呼叫患者」**发起（POC 用前端 `CallProvider` 总线跨端打通）；AI 仿真来电按日程/事件自动发起。两者患者侧都自动接通。

## 3. 视觉风格

- **基调**：温暖疗愈、家的感觉。暖色调（米白/暖橘/柔绿点缀），大圆角，柔和阴影，无硬边框。规避冷峻医疗器械感。
- **照护者控制台**：暖调底色上承载信息密度——卡片化、清晰层级、状态用颜色语义（绿=正常/已完成、琥珀=待处理/未服药、红=SOS/失联）。
- **患者屏**：超大字号（正文 ≥ 22px，按钮文字 ≥ 28px）、高对比、最少元素、单列、无横向滚动；每屏一个主任务；家人为「头像+名字」而非文字列表。
- **动效**：克制、缓和（淡入、轻微缩放），避免突兀闪烁惊扰患者。
- **图标 + 文字双标注**：患者屏所有可点元素都配文字，不靠纯图标。
- 参考图：以用户后续提供的参考图为准，本节定大方向。

## 4. 数据层与数据模型

所有页面经 `services/` 读写；当前实现读 `mock/`，含可配置的 `MOCK_DELAY` 与 `MOCK_FAIL_RATE` 以演示 loading/error。

核心实体（TypeScript 形态示意）：

```ts
Patient        { id, name, photo, age, lastSeenAt, status: 'online'|'offline', mood?: Mood }
Caregiver      { id, name, phone, relationToPatient }            // POC 仅一个

                 title, time, repeat, status:'pending'|'done'|'missed', voiceId? }
SOSEvent       { id, createdAt, state:'triggered'|'sent'|'acknowledged'|'resolved'|'escalated'|'cancelled',
                 lastLocation?: {lat,lng,label}, timeline: SOSStep[] }
MemoryCard     { id, name, relation, nickname, photo }
VoiceProfile   { id, memberName, relation, authorized:boolean, authorizedAt?,
                 sampleSeconds, learnProgress: number, status:'ready'|'recording'|'revoked' }
Conversation   { id, voiceId, startedAt, durationSec, summary, mood: Mood,
                 flaggedSensitive: boolean, transcript: TurnLine[] }
Mood           = 'calm'|'happy'|'anxious'|'confused'|'sad'
```

## 5. 照护者控制台 — 逐页规格

> 每页统一四态：**加载中 / 空数据 / 出错 / 成功**。下面只列各页特有内容。

### 5.1 概览 Dashboard `/caregiver`
- **核心任务**：进入即知患者「现在好不好」。
- **成功态**：顶部患者状态横幅（在线/失联 + 最后在线时间）；四张卡片：今日服药完成度、最近情绪、待处理告警数、最近一次 AI 对话。任一卡片可点进对应页。
- **加载中**：骨架屏（横幅 + 四卡占位）。
- **空数据**：尚无任何配置时，引导卡「先添加患者信息 / 第一条提醒 / 第一张认人卡」。
- **出错**：整页错误占位 + 重试按钮；若仅单卡数据失败，则该卡内联报错+重试，不拖垮整页。
- **特例**：患者「失联」时横幅置红，置顶且不可被其它卡覆盖。

### 5.2 提醒管理 `/caregiver/reminders`
- **核心任务**：管理用药/进餐/喝水/就医提醒，查看完成与错过。
- **成功态**：按时间排序的提醒列表，每条带状态徽标（待办/已完成/已错过）；右上「新增提醒」；可选绑定某家人声线播报。
- **加载中**：列表骨架。
- **空数据**：「还没有提醒，点此创建第一条」插画 + CTA。
- **出错**：列表区错误占位 + 重试。
- **交互**：新增/编辑用弹层表单（类型、标题、时间、重复、声线）；保存中按钮 loading，成功 toast，失败保留表单并提示。
- **特例**：「已错过」的提醒高亮琥珀色，并在告警中心同步出现一条升级记录（见 §8.1）。

### 5.3 告警 / SOS 中心 `/caregiver/alerts`
- **核心任务**：实时跟踪 SOS、未服药升级、失联，并留存历史。
- **成功态**：上方「进行中」区（活跃 SOS 卡，展示全链路时间线，见 §8.2）；下方「历史」列表。
- **加载中**：卡片骨架。
- **空数据**：「一切正常，暂无告警」安心态插画（绿色基调）。
- **出错**：错误占位 + 重试。
- **特例**：有进行中 SOS 时，全局给该入口加红点；活跃 SOS 卡支持「我已知晓 / 标记已解决」操作，状态即时流转。

### 5.4 认人卡管理 `/caregiver/memory-cards`
- **核心任务**：维护家人照片+姓名+关系+称呼，供患者屏认人。
- **成功态**：头像网格，每张可编辑/删除；「新增」卡。
- **加载中**：网格骨架。
- **空数据**：「添加第一位家人，帮助 TA 认出你」CTA。
- **出错**：错误占位 + 重试。
- **交互**：新增卡上传照片（POC 用本地占位图/预置图）、填称呼。保存四态同 §5.2。

### 5.5 声线管理 `/caregiver/voices`
- **核心任务**：授权录制家人声线、管理与撤销、展示 AI 学习占位。
- **成功态**：声线卡列表（家人名、关系、授权状态、样本时长、学习进度条占位）；每张可「撤销授权」。顶部强提示：**声线须经家人本人授权录制**（见 §9）。
- **加载中**：卡片骨架。
- **空数据**：「还没有授权任何声线」+「开始授权录制」CTA。
- **出错**：错误占位 + 重试。
- **交互·授权录制流程**：①勾选「我已获得本人同意」授权声明 → ②（mock）录制采样动画 → ③生成声线卡（状态 ready）。撤销需二次确认；撤销后该声线在患者屏立即不可用。
- **特例**：未授权/已撤销的声线，患者屏对应家人按钮置灰并标「暂不可对话」。

### 5.6 对话记录 `/caregiver/conversations`
- **核心任务**：把患者×AI 对话作为健康/情绪信号回看。
- **成功态**：按时间倒序的对话列表，每条显示家人声线、时长、情绪标签、是否含敏感话题标记；点开看摘要与逐句记录。
- **加载中**：列表骨架。
- **空数据**：「还没有对话记录」。
- **出错**：错误占位 + 重试。
- **特例**：被标记「敏感话题」（见 §9）的对话置顶并带醒目标记，提示照护者关注。

### 5.7 设置 `/caregiver/settings`
- **核心任务**：维护患者档案、主照护者信息、安全区占位。
- **成功态**：分区表单（患者基本信息、照护者联系方式、安全区域 mock 占位说明「真实定位不在 POC 范围」）。
- **四态**：加载骨架 / 首次为预置默认值（非空） / 保存出错内联提示 / 保存成功 toast。

## 6. 患者端「老人屏」（横屏大屏） — 逐页规格

> 设计前提：患者可能困惑、视力下降、易焦虑，设备为**固定摆放的横屏智能屏/电视**。核心是**零操作被动接收来电**；主动能力（拨出、认人）保留但不作为必经路径。超大字、高对比、每屏一个任务。

### 6.1 待机主页 `/patient`（横屏）
- **核心任务**：安心待机 + 随时被动接听 + 三件触手可及的事（和家人聊、看今天、求助）。
- **成功态**：左区超大问候（「下午好，张爷爷」）+ 巨型时钟/日期 + 今日下一条提醒卡 + **巨大红色 SOS**；右区（暖色分区）家人大头像可**主动拨出**、「认认人」入口，及演示用「模拟来电」触发。
- **被动来电**：收到来电时弹出全屏覆盖层（见 6.6），倒计时**自动接通**，患者无需任何操作。
- **加载中**：温和「正在准备…」大字 + 柔和动画。
- **空数据**：无家人/提醒时仍显示问候与 SOS；家人区「还没有家人哦」温和占位。
- **出错**：**绝不显示技术错误**，退化为「现在连接有点慢，先歇一会儿」+ SOS 仍可用。

### 6.2 通话页 `/patient/talk/:voiceId`（横屏）
- **核心任务**：被动接通后或主动拨出后，与家人通话（AI 仿真声线或真人）。
- **成功态**：左区大头像 + 名字 +「正在和女儿通话」+ **环境聆听指示「正在聆听…」（设备自动收音，无按住说话）** + 挂断；`?by=ai`（或主动拨出）显示「AI 模拟声音」明示，`by=family`（照护者发起的真人来电）不显示；右区大字幕对话 + 敏感话题守护提示。
- **加载中**：「正在接通家人…」拨号态动画。
- **出错**：「家人那边信号不太好，我们待会儿再聊」温和退化，回主页。
- **特例（敏感话题）**：患者说出「我要出门/我难受/找不到家」等，AI 先安抚，**不胡编**，并静默触发照护者通知 + 标记该对话敏感（见 §9）。

### 6.3 认人卡 `/patient/who`
- **核心任务**：帮患者认出身边的人。
- **成功态**：全屏大头像，下方大字「这是你的女儿 小雯」；左右滑或大箭头切换；可「和 TA 聊聊」直达对话页。
- **加载中**：温和过场。
- **空数据**：「还没有添加家人」温和占位 + 回家按钮。
- **出错**：温和退化 + 回家。

### 6.4 提醒弹窗（覆盖层）
- **核心任务**：到点用大字+（可选家人声线）提醒患者做某事。
- **成功态**：全屏覆盖，大图标+大字「该吃药啦」+（可选）女儿声线播报；两个大按钮「已完成 / 稍后提醒」。
- **交互**：点「已完成」→ 记录 done，关闭；点「稍后」→ 延后并稍后再弹；**超时未操作 → 标记 missed 并升级通知照护者**（§8.1）。

### 6.5 SOS 确认（覆盖层）
- **核心任务**：求助，但防误触。
- **成功态**：按 SOS 后进入**倒计时确认**（如 5 秒）「正在呼叫家人…（取消）」；倒计时结束自动发出，进入「已通知家人，别担心，待在原地」安心态。
- **交互**：倒计时内点「取消」→ 中止，记录为 `cancelled`，不打扰照护者（或仅留轻量日志，见 §8.3）。
- **出错**：发送通道（mock）失败也要安抚患者「正在联系家人」，并在后台重试 + 在照护者端如实标记发送异常。

### 6.6 来电覆盖层（被动接听）
- **核心任务**：来电时零操作被动接通，患者只看只听。
- **成功态**：全屏覆盖，大头像 + 「家人来电 / AI 模拟来电」+ 名字/关系「想和你说说话」+「无需操作，N 秒后自动接通…」倒计时；倒计时结束**自动进入通话页**。
- **交互**：默认自动接通；另留可选「以后再说」挂断（不强制操作）。
- **触发**：家人来电 = 照护者控制台「远程呼叫患者」发起；AI 来电 = 日程/事件自动发起。POC 用前端 `CallProvider` 总线跨端打通（真实中经推送/信令，不在 POC 范围）。

## 7. 关键端到端流程（用户完整路径）

### 流程 A — 照护者远程呼叫 → 患者大屏被动接听
1. 照护者控制台「陪伴 → 远程呼叫患者」→ 点「现在呼叫患者」（以家人身份发起）。
2. 患者大屏弹出来电覆盖层「家人来电 · 小雯」→ 倒计时**自动接通**（患者零操作）。
3. 进入横屏通话页，家人真人声线开场，「正在聆听…」环境收音；患者只看只听。
4. 结束「挂断」→ 回待机；生成一条 Conversation（含情绪标签）。

### 流程 A' — 患者主动和家人对话
1. 患者待机页 → 点家人大头像「小雯」拨出。
2. 进入通话页拨号态 → 接通 → 女儿 AI 声线开场（显示「AI 模拟声音」）。
3. 患者自然说话（设备自动收音「正在聆听…」，无按住说话；POC 用预置回应）→ AI 字幕+语音回应。
4. 结束「挂断」→ 回待机；生成一条 Conversation（含情绪标签）。
5. 照护者「对话记录」出现该条，情绪曲线更新，Dashboard「最近对话」卡刷新。

### 流程 B — 照护者新建用药提醒到患者完成
1. 照护者「提醒管理」→ 新增 → 选「用药」「19:00」「绑定女儿声线」→ 保存（成功 toast）。
2. 到点 → 患者屏弹出提醒（女儿声线播报）。
3. 患者点「已完成」→ 状态 done。
4. 照护者 Dashboard「今日服药完成度」与提醒列表同步更新。

### 流程 C — SOS 全链路（含升级）
1. 患者按巨大 SOS → 倒计时确认（可取消）。
2. 未取消 → `triggered → sent`；患者屏进入安心态。
3. 照护者告警中心「进行中」出现活跃 SOS 卡，全局红点。
4. 照护者点「我已知晓」→ `acknowledged`。
5. 若超时未响应 → `escalated`（POC 单照护者：升级表现为更醒目告警+重复提醒，不外呼）。
6. 照护者「标记已解决」→ `resolved`，进历史。

### 流程 D — 声线授权与撤销
1. 照护者「声线管理」→ 开始授权录制 → 勾选「已获本人同意」→ mock 采样 → 生成 ready 声线。
2. 患者屏对应家人变为可对话；学习进度条占位渐增。
3. 照护者「撤销授权」→ 二次确认 → 患者屏该家人置灰「暂不可对话」。

## 8. 异常路径细化（POC 必须演出）

> §8.1 与 §8.4 已由**真实状态机**实现：`src/state/monitor.tsx`（`MonitorProvider`）用计时器/事件驱动，跨页联动告警中心与 Dashboard，**非预置假数据**。告警中心底部「演示 · 异常路径」可手动触发（服药计时 / 断开患者），也会随计时自动发生。

### 8.1 提醒未完成升级
触发：提醒到点（`dueAt`）后仍 `pending` → 计时器置 `missed` 并**自动生成**「未服药」升级告警。表现：提醒页状态置红；告警中心历史新增高危记录；Dashboard 横幅「今日服药 X/Y」反映缺口。患者到点提醒「已完成」或提醒页「标记完成」即清除。

### 8.2 SOS 全链路状态
状态机：`triggered → sent → acknowledged → resolved`，分支 `→ escalated`（超时未响应）、`→ cancelled`（误触取消）。活跃 SOS 卡渲染完整时间线（每步带时间戳）。SOS 卡需展示**最后位置**（mock 坐标/静态地图占位，非真实 GPS）。

### 8.3 误触 SOS 可撤销
倒计时确认窗口内取消即中止，记 `cancelled`，默认不打扰照护者（仅轻量日志，告警中心不升级）。

### 8.4 患者设备离线/失联
判定：`connected=false` 后超过心跳阈值无心跳 → 计时器置 `online=false` 并**自动生成**失联告警。表现：Dashboard 与告警中心横幅置红「患者失联 · 最后在线 HH:MM」；「恢复在线」后置回在线并追加「已恢复在线」记录。由 `MonitorProvider` 计时器驱动，告警中心「断开患者/恢复在线」控制可演示。

### 8.5 单点数据失败不拖垮全局
任一 service 失败时，仅其所属区块显示内联错误+重试，其余区域正常；患者屏永不暴露技术错误，一律温和退化且保证 SOS 可用。

## 9. AI 仿声边界与安全（产品中必须体现）

1. **声线须家人授权录制**：录制前必须勾选本人同意声明；声线管理可随时撤销；撤销即时生效于患者屏。
2. **明示这是 AI**：对话界面（至少照护者可见视图与记录中）标注「AI 模拟声音」，不做完全欺骗。
3. **敏感话题转人工**：患者表达「要出门/难受/找不到家/想念已故亲人」等，AI 不胡编、先安抚，并触发照护者通知 + 将该对话标 `flaggedSensitive`。
4. **对话记录照护者可见**：所有患者×AI 对话生成摘要、情绪标签、逐句记录，供照护者回看，敏感对话置顶。

## 10. POC 明确不做（非目标）

- ⚠️ **真实语音克隆 / ASR / TTS**：录音为**真实麦克风采集**（浏览器 MediaRecorder，本地处理不上传）；「训练 + 合成」抽象在可插拔接口 `services/voiceApi.ts`（`enrollVoice/pollTraining/synthesize`），当前为 **mock 模拟进度**。接入真实后端（GPT-SoVITS / Replicate / HF 等）**只需替换该文件**——但真实推理需独立 GPU 后端，**不在当前静态部署范围**。对话回应仍为预置/规则。
- ❌ **真实账号 / 登录 / 鉴权**：角色切换为纯前端模拟，无注册、密码、真实身份。
- ❌ **推送 / 短信 / 电话 / 信令集成**：所有告警、SOS、来电通知仅在应用内模拟展示；跨端来电用前端 `CallProvider` 总线打通，不接真实推送/WebRTC/信令通道。
- ❌ **真实 GPS / 地图 SDK / 电子围栏**：定位相关一律 mock；安全区在设置里仅占位说明。
- ❌ **AI 真实持续学习/训练**：「学习进度」「已学到的偏好」仅为可视化占位。
- ❌ **多照护者 / 家庭群组 / 多级声线权限**：仅单一主照护者。
- ❌ **真实后端 / 数据库**：数据在 `mock/`，经 `services/` 抽象，未来可替换。

## 11. 端到端验证步骤（验收）

> 目标：一次性走通两端核心闭环 + 四态 + 关键异常路径。

**准备**：`npm install && npm run dev`，打开本地地址，确认顶部出现「照护者 ⇄ 患者」角色切换与演示控制台。照护者端为竖屏手机，患者端为横屏大屏。

1. **四态自检**（照护者）：用演示开关把「提醒管理」依次切到 加载中 / 空数据 / 出错（重试可恢复）/ 成功，逐一目视确认占位正确。
2. **流程 A（远程呼叫 → 被动接听）**：照护者「陪伴 → 远程呼叫患者 → 现在呼叫患者」→ 自动切到患者大屏，确认弹出「家人来电 · 小雯」覆盖层并**倒计时自动接通**（无需任何点击）→ 进入横屏通话页，确认为真人来电（**不显示**「AI 模拟声音」）、有「正在聆听…」环境指示 → 点「挂断」回待机。
3. **流程 A'（主动拨出 + AI 明示 + 敏感）**：患者待机页点家人头像「小雯」→ 接通，确认显示「AI 模拟声音」→ 大字幕对话含敏感话题（如「我想出门」）→ 确认 AI 先安抚不胡编 → 照护者「对话记录」出现该条并被标为敏感置顶，情绪标签可见。
4. **流程 B（提醒到点闭环）**：患者大屏触发「提醒到点」（演示按钮或 `?remind=1`）→ 确认全屏提醒覆盖层「该吃药啦」+ 女儿声线提示 + 大字「已完成 / 稍后提醒」→ 点「已完成」出现完成态；另验「稍后提醒」关闭，并确认文案提示"未操作将再次提醒并通知家人"（对应 §8.1 升级）。
5. **流程 C（SOS 全链路 + 误触）**：患者大屏按「紧急求助」→ 倒计时内点「取消」一次（记为 cancelled，照护者不升级）→ 再触发让倒计时结束进入「已通知家人」→ 照护者告警中心活跃卡出现，依次点「我已知晓」→「标记已解决」，确认时间线状态完整流转，最后位置以 mock 占位展示。
6. **异常·未服药升级 + 失联**：演示开关让一条提醒超时未完成 → 确认置 missed 且告警中心出现升级记录 → 触发「患者失联」→ 确认 Dashboard 横幅置红并显示最后在线时间 → 恢复心跳后告警自动消除。
7. **流程 D（声线授权/撤销）**：照护者授权一条女儿声线（勾选同意）→ 患者大屏该家人可对话 → 照护者撤销 → 确认患者大屏该家人即时置灰「暂不可对话」。
8. **患者屏韧性**：用演示开关强制患者端出错 → 确认患者大屏不暴露技术错误（温和退化「连接有点慢」）且「紧急求助」按钮仍可用。

全部通过即 POC 验收达标。
